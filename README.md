# 🏛️ 人文学科AI基础设施 (AIH Infra)

## AI Infrastructure for Humanities

**核心使命 (Core Mission)**：以未来 10 年乃至 30 年为架构周期，奠定文、史、哲学科AI研究的基础设施规范。  
**当前状态 (Current Status)**：v1.0 (Genesis) | **代号**: _Sapere Aude_ (敢于求知)

---

### 1. 宣言：从技术奇点走向人文深耕

**Manifesto: From Singularity to Deep Cultivation**

当下，人工智能（AI）正处于从爆发式的“技术奇点”向垂直领域的“人文深耕”转型的关键时刻。文、史、哲学科承载着理解传统、激发思辨、引导社会精神的重责。然而，数字鸿沟（Digital Divide）与工具壁垒（Tool Barriers）导致的人文学科AI应用体系混乱、重复造轮子、可复现性差，正在阻碍知识的深度积累与传承。

**AIH Infra** 应运而生。这不仅仅是一个代码仓库或文档集合，它是一项具有历史意义的奠基工程。我们致力于在两个看似对立的世界之间架起桥梁：

- **(Humanities Soul)**：守护人文温度，坚持每一页文献的尊严，追问技术背后的伦理与意义。
- **(Engineering Skeleton)**：奠定工程标准，确保每一个参数都有定义，每一个流程都可复现。

我们既关注高度的工程可靠性和科学复现，也坚持人文学科的批判性与深度思考，把“技术手册”与“思想宣言”合二为一。

### 2. 核心架构 (Core Architecture)

本基础设施由四个相互支撑的维度构成：

- 📜 **00-纲领 (Constitution)**：不可动摇的《白皮书》。它定义了我们为何出发，以及绝不逾越的底线——即“不可追溯，即非知识”。
- ⚖️ **01-立法 (Standards)**：可修订的数据标准。包括页码锚点规范（Page Anchor Protocol）、元数据 JSON Schema，以及针对人文材料的切块（Chunking）标准。
- 🔧 **02-工程 (Engineering)**：可执行的代码实现。从文献数字化（OCR Pipeline）到检索增强生成（RAG System），再到图谱推理（GraphRAG），提供开箱即用的工业级方案。
- 🧪 **03-实验室 (Benchmarks)**：可验证的评测基准。拒绝通用的跑分，建立专属于文史哲学科的“金标准”测试集，用数据检验模型在处理复杂人文语境时的真实能力。

### 3. 核心原则 (Core Principles)

|   |   |   |
|---|---|---|
|原则|英文|阐释|
|**可追溯性**|**Traceability**|**这是我们的最高法则。** 每一个AI生成的论断，必须能精确回溯到原始文献的版本与页码。没有页码的回答，只能算闲聊，不能算学术。|
|**可复现性**|**Reproducibility**|拒绝“黑箱”操作。从嵌入模型的选型到 Rerank 的参数，所有实验步骤必须透明公开，允许任何学者重现结果。|
|**版本意识**|**Versioning Awareness**|技术必须尊重文本的历史性。区分康德的A版与B版，区分底本与校本，绝不混淆不同时空的文本层级。|
|**人文优先**|**Humanities First**|技术是“替你翻书”的助手，绝非“替你思考”的主人。我们反对为了适应算法而削足适履地阉割人文材料。|
